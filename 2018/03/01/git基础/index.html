Posted on 2018-03-01 | Post modified 2018-03-08

暂存区

暂存区是指使用git add将文件暂存的空间，暂存的文件可以工作区的文件互相独立，比较暂存区或工作区相关的差异，有如下几种比较方式：

  • git diff –cached，比较暂存区的文件与最新一次提交的差异，即指定–cached参数来表明使用暂存区中的文件
  • git diff，比较暂存区和工作区两方的差异
  • git diff HEAD，比较工作区和最新一次提交的差异

基于提交历史进行版本间的比对、查询对于实际的开发跟踪有非常实用的价值，git log结合各种参数可以实现基于提交历史的查询
可以实用如下的一些参数来方便的显示、查询对应的提交历史

指定时间范围

假如要查询指定时间范围内的提交记录，可以使用如下参数：

指定作者

远程仓库

远程仓库与本地仓库相对应，远程仓库可以认为是本地仓库的远端存放空间，一个本地仓库可以存放在多个远端空间上，因而一个本地仓库可以对应多个远端仓库。在将本地仓库推送到远端仓库的时候，只需要指定远端仓库名称即可。

分支

git中的分支实质上是指向某个提交对象（commit）的指针，不同的分支名指向了各自对应的commit，git中HEAD指针来指向最新的一次提交

提交对象

git每次新建一个提交对象，会执行如下操作：

  • 为每个目录创建一个tree对象
    tree对象含有对应该目录的校验和，同时存放目录中各文件的blob对象的索引
  • 为每个文件创建blob对象
    git中文件的快照都是使用blob对象来存储的，每个文件对应一个blob对象，这些blob对象的索引被存放在各目录的tree对象中
  • 创建commit对象
    commit对象包含了指向tree对象的指针，以及创建该commit对象的一些元数据，例如提交的作者、时间、消息等

分支合并

分支合并通常有简单的三方合并和分支衍合

三方合并

该合并方式将两个分支的模块和它们的共同祖先进行一次三方合并计算，再将合并的结果作为一个新的提交。假设两个分支的共同祖先为A，两个分支的末端提交分别为B和C，三方合并的操作为对于已经跟踪的每个文件，将B和C中对应的版本与A中的版本文件的各个段落（或行）进行比较，操作逻辑如下：

  • 假如B和C中的内容与A中的内容一致，则该文件不作变更
  • 假如B中有变更而C中没有，则选取B中的变更到最新的快照中，反之亦然
  • 假如同一部分B和C都有变更，则会将两者的变更都放到最新的快照中，并将此处标记为冲突

衍合

三方合并在频繁使用分支进行开发且分叉众多的时候，这种合并操作很容易使得提交历史树的结构变得繁杂，使用衍合可以使得提交历史基本上都是在主线上一条直线下来，分叉更少。

衍合的工作流一般是从主分支取出一个特性分支，在本地完成开发后，且主分支更新不多的话，就可以在本地执行好衍合，再推送到远端仓库，开pull request让管理员合并到主分支
