hexo文件结构

完成网站的初始化后，会在blog文件中生成如下文件

各文件（夹）存放的内如大致为：

  • _config.yml
    用于配置hexo的一些全局参数
  • package.json
    用于配置hexo使用的一些js模块
  • scaffolds
    存放文章的模板，写文章的时候有三种类型，对应其中的三个文件名
  • source
    存放自己编写的原始文章，_drafts存放草稿，_posts存放需要发布的文章
  • themes
    存放主题文件夹，下载某个主题会在该文件夹中创建以该主题命名的子文件夹，要切换到某个主题的话，可以在_config.yml中的theme项中指定主题名即可
  • public
    存放hexo编译生成的网站文件，发布的时候就是将该文件夹推送到github仓库

github准备

在github上创建一个名为username.github.io的仓库，username即为自己的用户名。github会对与用户名同名的仓库特别对待，可以用来作为静态的网页空间
