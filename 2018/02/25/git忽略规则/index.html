Posted on 2018-02-25 | Post modified 2018-02-25

无效规则

.gitignore文件中的某些字符不会被git解析为忽略规则：

  • 空行
  • #开头的行（#开头的表示为注释）
  • 某行后面的空格会被忽略

shell通配符

官方叫法为shell glob，可以认为是一种简化的正则表达式，它的语法规则如下：

  • 星号（*）：匹配零个或任意多个字符
  • 问号（？）：匹配任意一个字符
  • [abc]：匹配方括号中的任意一个字符
  • [a-z]：短划线表示字符的范围，这里表示匹配a到z中的任意一个字符
  • [!] :匹配方括号内字符以外的任意字符，例如[!A]表示匹配除A以外的任意一个字符
